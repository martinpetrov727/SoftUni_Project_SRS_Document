# Task Board App (SoftUni Demo) – Manual Test Cases

## 📌 Overview
This project contains manual test cases for a Task Board web application developed as part of a SoftUni demo project.

The testing is based on the Software Requirements Specification (SRS) and focuses on validating core application functionality.

---

## 🎯 Scope
- Home page elements and navigation
- Task board structure and organization
- Task creation and validation
- Task viewing and editing
- Search functionality
- REST API access

---

## 🧪 Test Coverage
The test cases include:

- ✅ Positive scenarios (valid user actions)
- ❌ Negative scenarios (missing required fields)
- ⚠️ Validation checks (title, board selection)
- 🔍 Search functionality
- 🧩 UI structure and navigation
- 🔄 CRUD operations on tasks

---

## 🛠️ Approach
- Manual testing
- SRS-based validation
- Functional testing of core features
- Focus on user experience and system behavior

---

## 📂 Structure
Each test case includes:
- Test Case ID
- Title
- Preconditions
- Steps
- Expected Result
